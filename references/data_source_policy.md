# 数据源策略

## 参考依据

本策略参考 UZI-Skill README 的数据源设计：

- 实时行情 / PE / 市值：东方财富 push2，备源为雪球、腾讯、新浪、百度。
- 财报历史：AkShare，备源为雪球 F10。
- K 线 / 技术指标：AkShare。
- 龙虎榜 / 北向 / 两融：AkShare，备源为东方财富。
- 研报 / 公告：巨潮 cninfo + AkShare，备源为同花顺。
- 多层 fallback：一个源失败时自动切换下一个。

## 按数据类型选择主源

不要把单一数据源硬编码为所有字段的主源。按字段类型选择：

| 数据类型 | 主源 | 备源 |
|---|---|---|
| 实时行情、PE、市值、成交额 | 东方财富 push2 | 雪球 -> 腾讯 -> 新浪 -> 百度 |
| 财报历史 | AkShare | 雪球 F10 |
| K 线和技术指标 | AkShare `stock_zh_a_hist(..., adjust="qfq")` | AkShare 新浪 -> 新浪 K 线直连 -> 腾讯 ifzq K 线直连 |
| 龙虎榜、北向、两融 | AkShare | 东方财富 |
| 公告、研报 | 巨潮 cninfo + AkShare | 同花顺 |
| 宏观、政策、舆情、风险排查 | 可信搜索或官方页面 | 记录来源和查询时间 |

## 强制规则

- 不得混用不同日期的数据而不说明。
- 多源数据冲突时，按数据类型主源优先；主源不可用时按备源顺序降级，并记录 `data_sources` 和 `data_quality`。
- 备源只能补充真实字段，不能用于补造缺失字段。
- 若只有行情数据而缺少基本面、估值、公告，必须在结果中写入缺失字段并降低置信度。
- 每次筛选必须先生成 `outputs/{task_id}.signal.json`，通过校验后再生成报告。

## 推荐获取顺序

1. 使用东方财富 push2 获取全市场实时行情、PE、市值和成交额；失败时按雪球、腾讯、新浪、百度降级。
2. 过滤 ST、退市风险、停牌、成交额不足标的。
3. 使用 K 线 fallback 链补充候选池历史 K 线，计算趋势与量价因子；链路为 AkShare 东财 -> AkShare 新浪 -> 新浪 K 线直连 -> 腾讯 ifzq K 线直连。调用方式参考本地 `a-stock-quant/data/fetchers/history_fetcher.py` 和 UZI-Skill 的 K 线链路，但不读取 sqlite 缓存。
4. 使用 AkShare 获取财报历史；失败时尝试雪球 F10。
5. 使用巨潮 cninfo + AkShare 获取公告/研报；失败时尝试同花顺或记录缺失。
6. 记录每个字段的来源、可用日期、缺失字段和降级路径。

## 网络降级规则

- 若 3 个以上核心域名不可用，自动切换为 lite 模式，只做基础行情初筛。
- 若 push2 被反爬或连接关闭，立即降级到雪球；雪球需要登录或返回 4xx 时跳过到腾讯。
- 腾讯/新浪只能作为基础行情备源，不能替代完整基本面、估值和公告研究。
- 如果只能拿到基础报价，`data_quality.quality_score` 不应高于 0.5。
